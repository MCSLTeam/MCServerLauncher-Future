# WPF I18n Audit Plan

Date: 2026-06-16

## Touched Areas

- `frontend`: WPF XAML and user-facing dialog text.
- `workflow`: resource consistency checks, WPF build, and git hygiene.

## Goal

Scan the WPF project for user-facing hard-coded strings, replace confirmed UI text with `Lang.Tr[...]` resource bindings or lookups, and keep all locale resource files key-compatible.

## Tasks

- [x] Scan WPF XAML and C# for hard-coded user-facing strings.
- [x] Check resource key consistency across locale `.resx` files.
- [x] Localize confirmed Event Rule editor hard-coded text.
- [x] Localize disabled pre-create-instance tooltips.
- [x] Re-run i18n scans and decide whether remaining hits are user-facing strings, debug-only text, brands, protocol values, or format literals.
- [x] Build WPF, run protocol tests, and run final git hygiene checks.

## Changelog

- Added `check_i18n.ps1` support for scanning WPF XAML/C# `Lang.Tr` resource key usages and comparing missing keys across all six locale `.resx` files.
- Fixed daemon edit dialog resource lookup by replacing dynamic `Lang.Tr[...]` access with statically-scannable resource keys.
- Added `EditDaemon` translations across all six locale resources.
- Localized Event Rule editor ruleset labels, descriptions, execution-mode labels, and action separators.
- Localized disabled pre-create-instance tooltips with `FeatureNotReadyTip`.
- Re-ran the i18n scan after the targeted localization work; remaining Chinese string hits are outside this plan's scope, mainly language names, mirror paths, debug text, home-page copy, and File Manager UI text.
- Added `RULES.md` workflow requirements for plan usage, superpowers plan/subagent workflows, concise Conventional Commit titles, and plan changelog sections.
